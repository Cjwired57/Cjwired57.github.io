Arrays and Hashes

Hey, today I’ve decided to talk about two of the most common container classes in Ruby: arrays and hashes. What is a container object? Well, when you’re programming down the road, you will most likely find yourself dealing with not just one or two objects, but huge collections of objects. These collections are best represented by container objects like arrays and hashes.

Arrays

What is an array? Basically an array is just an ordered collection of objects…you can select one object from the collection based on it’s placement inside the collection. For example, let’s say you have an array of prime numbers, we’ll call it array_prime. 

 Array_prime = [2,3,5,7,13,17] 
  

    Now you can access any of these objects by it’s numerical index, or where it appears in the context of the collection as a whole.

    If I want to call a specific object, I can now just use:
  


 array_prime[0] 
  

    Which would return the first object in the array (a little confusing, but we start on 0 instead of 1).

    So if I wanted to access the number 13 I would do this:
  


 array_prime[4] 
  

    Arrays can hold virtually any object you want: strings, floats, integers, classes, other arrays, even hashes. Hashes, on the other hand, store objects in pairs, with each pair consisting of a key and a value.  So if I wanted to store some personal information about myself, I could set up a hash with values associated to keys…check it out:
  


 hash_personal_info = {“favorite-drink” => “Inca Kola”, “favorite-snack” => “funyuns”} 
  

    So right off the bat a few things have changed since we discussed arrays.  First of all, instead of grouping the objects in brackets as with arrays, we now group them in curly brackets, which are much more fun.

    Second, what is that “=>” doing there?  Basically => is an operator that assigns the value to the key.  So basically it is what tells the hash that funyuns are my favorite snack.  Now you can access these objects in a similar way as with arrays, using the bracket tools.
  

  

      If I wanted to access the value for favorite-drink, all I would do is:
  


 hash_personal_info[“favorite-drink”] 
  

    See? Pretty easy! It can also help to think of arrays as hashes where each of the keys are just consecutive numbers.  In the array_prime, we can think of the pairs almost as you would the x and y variables on a graph…(0,2), (1,3), (2,5), etc.  Where each key would be the x value and each value the y.
  

  

    Arrays and hashes are useful in a ton of different situations, and you’ll often have to use your best judgment on when to use either for storing information.  Arrays can be helpful when you are dealing with data that is related to one variable…like if I wanted to compare the height of each person in the class.  However, if I wanted to what each person’s favorite soda was, I might want to use hashes since the pairing gives me more customization.
